Rompiendo el hielo y empezando a conocer uno de los frameworks más poderosos y flexibles para Javascript

Por Alberto De la Hoz / @albertodlh

¿Qué es AngularJS?

  • Framework Javascript de lado de cliente.
  • Añade interactividad a HTML.
  • Open Source - Mantenido por Google.
  • Usa patrón de arquitectura MVC.
  • Utilizado para aplicaciones web dinámicas y de páginas únicas (single page apps).

Filosofía de AngularJS

  • Es buena idea dividir tareas entre el cliente y el servidor.
  • Lógica de la aplicación separada de la manipulación del DOM.
  • Las tareas comunes deben ser triviales (callbacks, manipulación del dom, código de inicialización).
  • Escribir tests debe ser tan importante como escribir la aplicación.

¿Por qué usar AngularJS?

  • Permite organizar a un equipo de devs en torno a un mismo paradigma.
  • Mantenido y promovido por Google.
  • jQuery (jqLite) "integrado".
  • No es necesario un template engine adicional.
  • ¡Porque es divertido!.

¿Por dónde empezamos?

Angular añade interactividad a HTML (No es necesario un template engine adicional).

Two way data binding

Angular sincroniza automáticamente los datos entre un modelo y sus componentes de visualización, manteniendo los datos sincronizados donde quiera que se visualicen.

Controladores

Puente que vincula los datos de un modelo con una vista. Nos permite añadir funcionalidad más compleja a una aplicación.

Directivas

Atributos, clases, comentarios o elementos HTML a los que se asocian comportamientos. Se utilizan cuando es necesario realizar manipulación compleja del DOM.

Servicios

Instancias únicas de objetos (singletons) utilizados para organizar y compartir código y datos a través de la aplicación. Pueden agregarse como dependencias de controladores, directivas, o de otros servicios.

¿Services, factories o providers?

  • Todos son singletons, pero...
  • Los servicios se definen con funciones constructoras. El singleton es creado usando el operador new sobre la función.
  • Las factories se definen con funciones que retornan un objeto de JS. El singleton se crea ejecutando la función.
  • Los providers se definen con funciones construtoras (igual que los servicios). La diferencia es que son el único tipo de servicio que puede inyectarse en la función .config() de la aplicación.

Componentes

Existen desde Angular 1.5 y son un tipo especial de directiva, más simple de configurar, y que hace mucho más fácil la migración de una aplicación Angular 1.x a Angular 2 (arquitectura basada en componentes).
